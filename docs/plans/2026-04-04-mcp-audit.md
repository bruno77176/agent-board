## 🐛 Bugs

### BUG-1 (HIGH) — `complete_story`: boolean param rejected as string · BOARD-19
`checklist_confirmed: true` is passed as the string `"true"` by the MCP tool interface. Zod `z.boolean()` rejects it.

**Fix:** Use `z.preprocess(v => v === 'true' || v === true, z.boolean())` in the Zod schema, or use `z.coerce.boolean()`.

---

### BUG-2 (HIGH) — `create_feature` / `create_story`: array and number params rejected · BOARD-20
- `tags: ["auth"]` → `Expected array, received string`
- `estimated_minutes: 5` → `Expected number, received string`

**Root cause:** MCP SDK serializes all tool parameters as strings before passing them to the handler. Affects any Zod schema using `z.array()` or `z.number()` directly.

**Fix:** Add `z.preprocess` coercion for affected fields:
```ts
tags: z.preprocess(
  v => typeof v === 'string' ? JSON.parse(v) : v,
  z.array(z.string()).optional()
)
estimated_minutes: z.coerce.number().optional()
```

---

### BUG-3 (MEDIUM) — `delete_story_link`: crashes with "Unexpected end of JSON input" · BOARD-21
The DELETE endpoint returns HTTP 204 with no body. `board.ts` calls `res.json()` unconditionally, crashing when the body is empty.

**Fix:** In `board.ts`, guard JSON parsing:
```ts
const text = await res.text()
return text ? JSON.parse(text) : null
```

---

### BUG-4 (MEDIUM) — Write operations reject short_ids · BOARD-22
`start_story('BOARD-1', ...)` returns 404. All write endpoints require UUIDs. Read endpoints (`get_epic`, `get_story`) already accept short_ids — write endpoints should too.

**Fix:** In each write route, resolve short_id → UUID before operating:
```ts
const row = db.prepare('SELECT id FROM stories WHERE id = ? OR short_id = ?').get(id, id)
```

## 💡 Missing Features / Enhancements

### MISSING-1 (MEDIUM) — `update_feature` tool · BOARD-23
No way to edit a feature's title, description, or tags after creation. `update_story` and `update_epic` exist — feature is the gap.

---

### MISSING-2 (MEDIUM) — Delete tools for story / feature / epic · BOARD-24
No `delete_story`, `delete_feature`, or `delete_epic`. Agents cannot remove erroneous entities. Critical for iterative workflows and test cleanup.

---

### MISSING-3 (MEDIUM) — `list_stories` with filters · BOARD-25
Stories are only accessible via `get_board` (all) or `get_feature` (by feature). No way to query:
- Stories in a specific status across a project
- Stories assigned to a specific agent
- Stories matching a tag

Needed for agent dashboards ("what's my work queue?").

---

### MISSING-4 (LOW) — Doc watcher requires filesystem access · BOARD-26
The doc watcher monitors `DOCS_PATH` on the server. Against a deployed Railway instance, agents cannot write files. An HTTP endpoint (`POST /api/docs/sync` with markdown body) would make doc-driven board population work remotely.

---

### MISSING-5 (LOW) — No way to create public projects via MCP · BOARD-27
`create_project` always creates private projects (`is_public=0`). No MCP tool exposes the `is_public` flag or a `make_project_public` operation.
